# Техническое задание на проект «Тестирование веб-приложения Swag Labs»

## 1. Цель и задачи сайта
**Цель**:  
Предоставить демонстрационную e-commerce платформу для отработки навыков тестирования (ручного и автоматизированного) без необходимости разворачивать собственный сервер.

**Задачи сайта** (с точки зрения создателей — Sauce Labs):
- Демонстрация типичного пользовательского пути в интернет-магазине
- Возможность тестировать авторизацию с разными ролями/состояниями пользователей
- Практика работы с UI-элементами: таблицы, фильтры, формы, модальные окна, валидация
- Тренировка негативных сценариев (заблокированный пользователь, глюки производительности, ошибки UI)

## 2. Целевая аудитория
- Студенты и начинающие QA-инженеры
- Автоматизаторы (те, кто изучает Selenium, Cypress, Playwright, Appium и т.п.)
- Тренеры и преподаватели курсов по тестированию
- Специалисты, готовящиеся к собеседованиям (очень часто просят протестировать/написать автотесты именно для saucedemo.com)

## 3. Проблемы, которые решает приложение
- Отсутствие стабильного, бесплатного и всегда доступного полигона для практики автоматизации
- Сложность поиска реального e-commerce сайта без риска забанить аккаунт или нарушить ToS
- Необходимость отрабатывать разные пользовательские сценарии и ошибки в одном месте
- Возможность быстро начать проект без установки зависимостей и деплоя

## 5. Описание работы приложения
- **Тип проекта**: веб-приложение (browser-based)
- **Технологии**: HTML, CSS, JavaScript (React-подобный фреймворк)
- **Основные экраны / user flow**:
  1. Login page
  2. Inventory / Products (список товаров, сортировка, добавление в корзину)
  3. Cart
  4. Checkout: Your Information
  5. Checkout: Overview
  6. Checkout: Complete (подтверждение заказа)

- [ ] Открыть страницу авторизации

- [ ] Нажать кнопку "Войти"

- [ ] Проверить, что изменения были сохранены  
- [ ] Проверить, что отображается правильная информация о пользователе
