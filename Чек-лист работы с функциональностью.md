# Чек-лист: Работа с функциональностью (основные сценарии)

## 1. Поиск, фильтры и пагинация (адаптировано под SauceDemo)
- [ ] **Сортировка товаров (Filter):**
  - [ ] Выпадающий список сортировки содержит опции: Name (A to Z), Name (Z to A), Price (low to high), Price (high to low)
  - [ ] При выборе "Name (A to Z)" товары упорядочиваются по алфавиту
  - [ ] При выборе "Name (Z to A)" — в обратном порядке
  - [ ] При выборе "Price (low to high)" — по возрастанию цены
  - [ ] При выборе "Price (high to low)" — по убыванию цены
- [ ] **Пагинация:** в приложении отсутствует (все товары на одной странице) — N/A
- [ ] **Поиск:** функциональность поиска не реализована — N/A

## 2. Работа кнопок и ссылок
- [ ] **Кнопки навигации:**
  - [ ] Кнопка "Back to products" на детальной странице ведёт обратно в каталог
  - [ ] Кнопка "Continue Shopping" в корзине возвращает в каталог
  - [ ] Кнопка "Checkout" в корзине ведёт на страницу информации о покупателе
  - [ ] Кнопка "Cancel" на шаге оформления возвращает в корзину или каталог (соответственно)
  - [ ] Кнопка "Finish" завершает покупку и показывает страницу подтверждения
  - [ ] Кнопка "Back Home" на странице завершения возвращает в каталог
- [ ] **Ссылки в боковом меню:**
  - [ ] "All Items" — переход в каталог
  - [ ] "About" — переход на внешний сайт Sauce Labs (открывается в новой вкладке)
  - [ ] "Logout" — разлогинивание и переход на страницу логина
  - [ ] "Reset App State" — сброс состояния (корзина очищается)
- [ ] **Кнопки добавления/удаления товаров:**
  - [ ] "Add to cart" работает на всех страницах
  - [ ] "Remove" удаляет товар из корзины и обновляет интерфейс

## 3. Переходы между страницами
- [ ] **Авторизация → Каталог:** после успешного логина открывается страница со списком товаров
- [ ] **Каталог → Детальная страница:** клик по названию или изображению товара ведёт на его карточку
- [ ] **Каталог → Корзина:** клик по иконке корзины ведёт на страницу корзины
- [ ] **Корзина → Оформление заказа:** последовательные шаги Checkout: информация → обзор → завершение
- [ ] **Оформление заказа → Каталог:** после завершения или отмены возврат в каталог
- [ ] **Боковое меню → Logout:** переход на страницу логина
- [ ] **Прямые URL-переходы:**
  - [ ] Ввод URL страницы корзины без авторизации — редирект на логин
  - [ ] Ввод URL каталога после логаута — редирект на логин

## 4. Соответствие требованиям (описанию в Техническом задании)
- [ ] **Пользователи с разными ролями:** проверить поведение для standard_user, locked_out_user, problem_user, performance_glitch_user, error_user, visual_user (описано в README)
- [ ] **Пароль:** для всех пользователей единый пароль `secret_sauce`
- [ ] **Основной сценарий покупки:** добавление товара → оформление заказа → подтверждение — выполняется без ошибок
- [ ] **Обработка ошибок:** при неверном логине/пароле появляется сообщение об ошибке, заблокированный пользователь не может войти
- [ ] **Валидация полей на шаге информации:** пустые поля не позволяют продолжить, выводятся сообщения об ошибках
